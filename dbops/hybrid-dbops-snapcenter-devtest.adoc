= 开发/测试爆发至云端的工作流程
:hardbreaks:
:allow-uri-read: 
:nofooter: 
:icons: font
:linkattrs: 
:imagesdir: ../media/


[role="lead"]
对于采用公共云进行数据库应用程序开发和测试工作的企业来说，公共云的灵活性、价值实现时间和成本节省都是有意义的价值主张。没有比SnapCenter更好的工具来实现这一目标。 SnapCenter不仅可以在本地保护您的生产数据库，还可以在公共云中快速克隆副本以用于应用程序开发或代码测试，同时仅消耗很少的额外存储空间。以下是使用此工具的逐步过程的详细信息。



== 从复制的快照备份克隆 Oracle 数据库以用于开发/测试

. 使用 Oracle 的数据库管理用户 ID 登录SnapCenter 。导航到“资源”选项卡，其中显示受SnapCenter保护的 Oracle 数据库。
+
image:snapctr-ora-clone-001.png["该图显示输入/输出对话框或表示书面内容"]

. 单击目标本地数据库名称以获取备份拓扑和详细视图。如果启用了辅助复制位置，则会显示链接的镜像备份。
+
image:snapctr-ora-clone-002.png["该图显示输入/输出对话框或表示书面内容"]

. 单击镜像备份即可切换到镜像备份视图。然后显示辅助镜像备份。
+
image:snapctr-ora-clone-003.png["该图显示输入/输出对话框或表示书面内容"]

. 选择要克隆的镜像辅助数据库备份副本，并通过时间和系统更改号或 SCN 确定恢复点。通常，恢复点应该落后于要克隆的完整数据库备份时间或 SCN。确定恢复点后，必须安装所需的日志文件备份以进行恢复。日志文件备份应安装到要托管克隆数据库的目标数据库服务器。
+
image:snapctr-ora-clone-004.png["该图显示输入/输出对话框或表示书面内容"]

+
image:snapctr-ora-clone-005.png["该图显示输入/输出对话框或表示书面内容"]

+

NOTE: 如果启用了日志修剪并且恢复点超出了最后一次日志修剪，则可能需要安装多个存档日志备份。

. 突出显示要克隆的完整数据库备份副本，然后单击克隆按钮以启动 DB 克隆工作流。
+
image:snapctr-ora-clone-006.png["该图显示输入/输出对话框或表示书面内容"]

. 为完整的容器数据库或 CDB 克隆选择适当的克隆 DB SID。
+
image:snapctr-ora-clone-007.png["该图显示输入/输出对话框或表示书面内容"]

. 选择云中的目标克隆主机，克隆工作流创建数据文件、控制文件和重做日志目录。
+
image:snapctr-ora-clone-008.png["该图显示输入/输出对话框或表示书面内容"]

. None 凭证名称用于基于操作系统的身份验证，这使得数据库端口无关紧要。按照目标克隆数据库服务器中的配置填写正确的 Oracle Home、Oracle OS User 和 Oracle OS Group。
+
image:snapctr-ora-clone-009.png["该图显示输入/输出对话框或表示书面内容"]

. 指定克隆操作之前要运行的脚本。更重要的是，可以在这里调整或定义数据库实例参数。
+
image:snapctr-ora-clone-010.png["该图显示输入/输出对话框或表示书面内容"]

. 通过日期和时间或 SCN 指定恢复点。直到取消将数据库恢复到可用的存档日志。从挂载存档日志卷的目标主机指定外部存档日志位置。如果目标服务器 Oracle 所有者与内部生产服务器不同，请验证目标服务器 Oracle 所有者是否可以读取存档日志目录。
+
image:snapctr-ora-clone-011.png["该图显示输入/输出对话框或表示书面内容"]

+
image:snapctr-ora-clone-012.png["该图显示输入/输出对话框或表示书面内容"]

. 如果需要，配置 SMTP 服务器以接收电子邮件通知。
+
image:snapctr-ora-clone-013.png["该图显示输入/输出对话框或表示书面内容"]

. 克隆摘要。
+
image:snapctr-ora-clone-014.png["该图显示输入/输出对话框或表示书面内容"]

. 您应该在克隆后进行验证，以确保克隆的数据库可以运行。可以在 dev/test 数据库上执行一些额外的任务，例如启动监听器或者关闭 DB 日志归档模式。
+
image:snapctr-ora-clone-015.png["该图显示输入/输出对话框或表示书面内容"]





== 从复制的快照备份克隆用于开发/测试的 SQL 数据库

. 使用 SQL Server 的数据库管理用户 ID 登录SnapCenter 。导航到“资源”选项卡，该选项卡显示受SnapCenter保护的 SQL Sever 用户数据库和公共云中的目标备用 SQL 实例。
+
image:snapctr-sql-clone-001.png["该图显示输入/输出对话框或表示书面内容"]

. 单击目标本地 SQL Server 用户数据库名称以获取备份拓扑和详细视图。如果启用了辅助复制位置，则会显示链接的镜像备份。
+
image:snapctr-sql-clone-002.png["该图显示输入/输出对话框或表示书面内容"]

. 单击“镜像备份”切换到“镜像备份”视图。然后显示辅助镜像备份。由于SnapCenter将 SQL Server 事务日志备份到专用驱动器以进行恢复，因此这里仅显示完整的数据库备份。
+
image:snapctr-sql-clone-003.png["该图显示输入/输出对话框或表示书面内容"]

. 选择一个备份副本，然后单击“克隆”按钮启动“从备份克隆”工作流程。
+
image:snapctr-sql-clone-004-a.png["该图显示输入/输出对话框或表示书面内容"]

+
image:snapctr-sql-clone-004.png["该图显示输入/输出对话框或表示书面内容"]

. 选择云服务器作为目标克隆服务器、克隆实例名称、克隆数据库名称。选择自动分配挂载点或用户定义的挂载点路径。
+
image:snapctr-sql-clone-005.png["该图显示输入/输出对话框或表示书面内容"]

. 通过日志备份时间或特定日期和时间确定恢复点。
+
image:snapctr-sql-clone-006.png["该图显示输入/输出对话框或表示书面内容"]

. 指定在克隆操作之前和之后运行的可选脚本。
+
image:snapctr-sql-clone-007.png["该图显示输入/输出对话框或表示书面内容"]

. 如果需要电子邮件通知，请配置 SMTP 服务器。
+
image:snapctr-sql-clone-008.png["该图显示输入/输出对话框或表示书面内容"]

. 克隆摘要。
+
image:snapctr-sql-clone-009.png["该图显示输入/输出对话框或表示书面内容"]

. 监控作业状态并验证目标用户数据库是否已附加到云克隆服务器中的目标 SQL 实例。
+
image:snapctr-sql-clone-010.png["该图显示输入/输出对话框或表示书面内容"]





== 克隆后配置

. 本地的 Oracle 生产数据库通常以日志存档模式运行。对于开发或测试数据库来说，此模式不是必需的。要关闭日志存档模式，请以 sysdba 身份登录 Oracle DB，执行日志模式更改命令，然后启动数据库进行访问。
. 配置 Oracle 侦听器，或将新克隆的数据库注册到现有的侦听器以供用户访问。
. 对于 SQL Server，将日志模式从“完整”更改为“简单”，以便 SQL Server dev/test 日志文件在填满日志卷时可以轻松缩小。




== 刷新克隆数据库

. 删除克隆的数据库并清理云数据库服务器环境。然后按照前面的步骤克隆一个包含新数据的新数据库。克隆一个新数据库只需要几分钟。
. 关闭克隆数据库，使用 CLI 运行克隆刷新命令。有关详细信息，请参阅以下SnapCenter文档：link:https://docs.netapp.com/us-en/snapcenter/protect-sco/task_refresh_a_clone.html["刷新克隆"^] 。
