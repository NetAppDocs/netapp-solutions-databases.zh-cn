= Google Cloud NetApp Volumes 工作流，用于部署高可用性 Oracle Database 26ai
:hardbreaks:
:allow-uri-read: 
:nofooter: 
:icons: font
:linkattrs: 
:imagesdir: ../media/


[role="lead"]
使用 Google Cloud NetApp Volumes iSCSI 块存储在 Google Compute Engine 上部署具有 Oracle Data Guard 的高可用性 Oracle Database 26ai。此工作流程引导您完成部署选项、基础架构配置、Oracle 软件安装和高可用性配置，利用 GCNV 的存储层复制来加速备用初始化。

.image:https://raw.githubusercontent.com/NetAppDocs/common/main/media/number-1.png["一个"]link:gcnv-oracle-solution-architecture.html["查看解决方案架构"]
[role="quick-margin-para"]
了解适用于 Oracle Database 26ai 高可用性的 Google Cloud NetApp Volumes 架构，包括 GCNV iSCSI 块存储设计、Data Guard 集成和存储拓扑。

.image:https://raw.githubusercontent.com/NetAppDocs/common/main/media/number-2.png["两个"]link:gcnv-oracle-deployment-options.html["查看部署选项和要求"]
[role="quick-margin-para"]
比较从开发/测试到生产 HA 的部署层，并确认基础架构、访问和软件先决条件。

.image:https://raw.githubusercontent.com/NetAppDocs/common/main/media/number-3.png["三"]link:gcnv-oracle-access-and-apis.html["验证访问和 API"]
[role="quick-margin-para"]
确认所需访问、API 启用、网络先决条件和时间同步。

.image:https://raw.githubusercontent.com/NetAppDocs/common/main/media/number-4.png["四个"]link:gcnv-oracle-provision-compute.html["预配 Google Compute Engine 实例"]
[role="quick-margin-para"]
为主数据库、备用数据库和 Fast-Start Failover Observer 创建虚拟机。

.image:https://raw.githubusercontent.com/NetAppDocs/common/main/media/number-5.png["五"]link:gcnv-oracle-provision-storage.html["预配 GCNV iSCSI 存储"]
[role="quick-margin-para"]
创建 GCNV 存储池和 iSCSI 卷，配置多路径，并设置 ASM 磁盘组。

.image:https://raw.githubusercontent.com/NetAppDocs/common/main/media/number-6.png["六"]link:gcnv-oracle-install-software.html["安装Oracle Grid Infrastructure和Oracle Database 26ai"]
[role="quick-margin-para"]
在两个数据库主机上安装 Oracle Grid Infrastructure 和 Oracle Database 26ai 软件。

.image:https://raw.githubusercontent.com/NetAppDocs/common/main/media/number-7.png["七"]link:gcnv-oracle-primary-database.html["创建主数据库"]
[role="quick-margin-para"]
使用 ASM 存储创建主 Oracle 数据库并配置 Oracle Restart。

.image:https://raw.githubusercontent.com/NetAppDocs/common/main/media/number-8.png["八个"]link:gcnv-oracle-standby-database.html["创建备用数据库"]
[role="quick-margin-para"]
使用 GCNV 复制、快照或克隆初始化备用数据库，然后向 Oracle Restart 注册。所有 HA 层均完成此步骤。

.image:https://raw.githubusercontent.com/NetAppDocs/common/main/media/number-9.png["九"]link:gcnv-oracle-standby-finalize-dg.html["为 Data Guard 完成备用数据库"]
[role="quick-margin-para"]
创建备用重做日志，启用闪回，激活重做传送，并验证 Data Guard 状态。*特定层级：* 仅适用于 Prod HA（Data Guard + FSFO）层级。

.image:https://raw.githubusercontent.com/NetAppDocs/common/main/media/number-10.png["十"]link:gcnv-oracle-data-guard.html["配置 Data Guard Broker 和 Fast-Start Failover"]
[role="quick-margin-para"]
启用 Broker、创建 Data Guard 配置、启用 FSFO 并配置 Observer。*特定于层级：* 仅适用于生产 HA（Data Guard + FSFO）层级。
